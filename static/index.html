API Description


The Wordslang API makes it easy to discover different representations of words in the English language.

Wordslang uses a REST(ish) API, supporting GET and a limited number of experimental POST functions.

All responses are in JSON format.



Wordslang Queries


Resource URL


GET http://word-redsquirrel.dotcloud.com/wordslang


Returns


All simple queries return a list of dictionaries (in pythonic) so that multiple matches can be returned. Even when only a single value is found, it is returned in a list.


Parameters

ParameterDescriptionAcceptable Values
token
required		 The token to search for, irrespective of the field being searched Any word, phrase, slang, phonetic representation or emoticon
check
required		 The field to look for 'token' in all, word, slang, emo
output
required		 The field to be returned word, slang, pho, exists (see examples), all (returns all previous)
regex
optional		 Submits the word with wildcards at the start and end of the token being submitted. Experimental. And slow. boolean
distinct
optional		 Only available with check=pho to return a unique list of phonetics. boolean


Adding Emoticons


Resource URL


POST or PUT (Experimental) http://word-redsquirrel.dotcloud.com/postemo


Payload


A list/array of {word : emoticon} as a JSON string


Example: [{"smile":":-)"},{"smile",":-]"}]



Adding Slang


Resource URL


POST or PUT (Experimental) http://word-redsquirrel.dotcloud.com/postslang


Payload


A list/array of {word : slang} as a JSON string


Example: [{"hello":"hi"},{"hello","yo"}]
